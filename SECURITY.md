## Security Policy

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Instead, report security concerns privately to the maintainers with:
- Affected files/modules
- Reproduction steps
- Potential impact
- Suggested mitigation (if available)

You will receive an acknowledgment and follow-up once triage is complete.

## Scope

This policy applies to repository source code and tooling. It does not grant permission to test external systems or protected clinical infrastructure.
